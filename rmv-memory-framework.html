RMV Memory Framework

A reusable CLI that turns project memory into something you can generate, not rebuild.

The RMV Memory Framework takes the manual memory pattern from V1 and makes it repeatable. A terminal CLI scaffolds the memory structure for a new project, and configurable variables adapt that structure to each project — while keeping the same human-in-the-loop execution discipline.

It turns AI-assisted work from temporary chat context into persistent, auditable project memory — now generated in one command instead of assembled by hand.

The CLI scaffolds the memory. The workflow governs the execution. The framework does not try to make AI fully autonomous — it makes AI-assisted execution structured, traceable, and human-reviewed.

update
Step 7

Memory is updated

Progress, decisions, and new context are written to persistent files.

gavel Why the Gates Exist

Each gate in this workflow came from a real failure — not from theory.

Gate 1 — The File Truncation Incident

During Voice-RMV development, the agent was asked to update progress.md. Without a plan-first rule, it overwrote the entire file with a partial summary — erasing weeks of validated progress entries in one write operation.

Rule created:

File Truncation Prevention — read the complete file before updating. Never rewrite blindly.

The gates are not bureaucratic overhead. They are scars from real mistakes — encoded as rules so the same failure does not happen twice.

05 / REUSABLE ASSETS

Plug-and-Play Bank

Plug-and-Play Bank transforms repeated workflow patterns into reusable operational assets — making delivery faster, more consistent, and easier to control.

Problem: Repeated Work

Most AI-assisted projects fail not because the model can't generate output. They fail because the same operational patterns are rebuilt from scratch every time: task plans, review checkpoints, debug notes, execution logs, and proof reports.

Solution: Plug-and-Play Bank

The Plug-and-Play Bank is the reusable asset layer connected to the RMV Memory Framework. It stores proven workflow patterns and operational building blocks that can be reused across different projects without exposing the entire internal implementation. Instead of starting from a blank process, each new project can inherit structured ways to plan tasks, request human review, document issues, log execution, and prove results.

This turns repeated AI-assisted work into reusable operational assets — making delivery faster, more consistent, and easier to control.

07 / OPERATIONAL TRUTH
